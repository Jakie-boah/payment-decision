Payment Decision

Асинхронный сервис обработки платежей с гарантированной доставкой событий (Outbox pattern).

📌 Описание

Сервис отвечает за:

создание платежей
обработку статусов
отправку событий через брокер сообщений
уведомление внешних систем через webhook

🏗 Технологии
   Python 3.13
   FastAPI
   PostgreSQL
   RabbitMQ
   SQLAlchemy (async)
   Alembic
   FastStream
   pytest

🚀 Быстрый старт

1. Установка зависимостей
   uv sync
2. Запуск приложения
   uv run just up

📌 При запуске автоматически:

применяются миграции
поднимается инфраструктура
стартует API

🧪 Тестирование
uv run just test



⚙️ Основные команды

# запуск проекта

uv run just up

# тесты

uv run just test


📁 Структура проекта
src/
├── domain/ # бизнес-логика и сущности
├── application/ # use-cases
├── infrastructure/ # БД, брокеры, интеграции
├── presentation/ # API слой, точки входа

🔄 Архитектура
Полностью асинхронная
Используется Outbox pattern для надежной доставки событий
Четкое разделение слоев (DDD-ish)

📡 Webhook

После обработки платежа сервис отправляет HTTP-запрос на указанный URL:

метод: POST
тело: JSON с данными платежа

✅ Особенности
идемпотентность платежей
высокая отказоустойчивость
покрытие тестами
готов к масштабированию
